# Team Rules (Android Studio / GitHub)

## Branch naming
- `feature/<short-name>`
- `fix/<short-name>`
- `chore/<short-name>`

Ví dụ:
- `feature/search`
- `feature/create-room`
- `fix/crash-on-submit`

## Commit messages (Conventional Commits)
Format:

`<type>(<scope>): <mô tả ngắn>`

Ví dụ:
- `feat(search): add SearchView filter rooms`
- `fix(adapter): correct status color`
- `docs(pr): add pull request template`

Allowed `type`: `feat`, `fix`, `docs`, `refactor`, `chore`

## Pull Request
- **Title**: `[Part X] <summary>`
- **Nội dung**: summary + how to test + screenshot (nếu có) + checklist

Tham khảo thêm: `CONTRIBUTING.md`
